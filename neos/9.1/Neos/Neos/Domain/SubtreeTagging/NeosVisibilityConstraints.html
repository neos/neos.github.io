final class NeosVisibilityConstraints (View source)

Neos specific content repository visibility constraints {VisibilityConstraints}

The visibility constraints define a context in which the content graph is accessed.

By default, Neos provides two kinds of subtree tags:

  • disabled {\Neos\Neos\Domain\SubtreeTagging\NeosSubtreeTag::disabled()} which is used when disabling a node
  • removed {\Neos\Neos\Domain\SubtreeTagging\NeosSubtreeTag::removed()} which is used when soft removing a node

To control which nodes will be queried via the {\Neos\ContentRepository\Core\Projection\ContentGraph\ContentSubgraphInterface}, the visibility constraints can be used.

The definitional here allow to implement the "frontend" and "backend" rendering use cases.

To access all nodes, including soft removed nodes, please use {[\Neos\ContentRepository\Core\Projection\ContentGraph\VisibilityConstraints::createEmpty()}

Specifying](../../../../Neos/ContentRepository/Core/Projection/ContentGraph/VisibilityConstraints.html) the visibility constraints manually is only necessary when breaking away from the by default provided constraints via {\Neos\Neos\Domain\SubtreeTagging\ContentRepository::getContentSubgraph()}. Circumventing the neos auth provider leads to no ReadNodePrivilege's being evaluated. All custom tagged nodes are visible at all times unless excluded via {\Neos\ContentRepository\Core\Projection\ContentGraph\VisibilityConstraints::merge()}
